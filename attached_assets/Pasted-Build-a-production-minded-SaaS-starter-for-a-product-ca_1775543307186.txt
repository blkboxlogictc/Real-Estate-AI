Build a production-minded SaaS starter for a product called “Estate Assistant AI” using React + TypeScript for the frontend, Node.js + TypeScript for the backend, and Supabase for auth/database/storage. The app should be structured so it can be exported cleanly from Replit and continued in VS Code. This is a multi-tenant SaaS for real estate agents to create and manage an AI voice assistant that answers calls, collects lead information, answers questions based on agent-specific data, books appointments, and sends summaries to the agent.

GENERAL GOAL
Create a clean MVP starter with a polished dashboard UI, strong file structure, placeholder integration modules, onboarding flow, and database-backed settings pages. Do not try to fully complete all external integrations in one shot, but wire the architecture correctly so the pieces are ready to connect. Prioritize maintainability, modularity, and clear separation between frontend, backend, database, and third-party integrations.

TECH STACK
- Frontend: React + TypeScript + Vite
- Styling: Tailwind CSS
- UI: clean modern SaaS dashboard style, professional and minimal
- Backend: Node.js + TypeScript with Express
- Database/Auth: Supabase
- State/data fetching: React Query
- Forms: React Hook Form + Zod validation
- Routing: React Router
- Environment variable support for all integrations
- Use a monorepo-style or clearly separated structure if possible, but keep it Replit-friendly

IMPORTANT ARCHITECTURE RULES
- The backend is the control center for all business logic
- The AI voice layer should NOT hold the business logic
- External services (Twilio, Vapi, Google Calendar, OpenAI, MLS/IDX/RESO connectors) should each have their own service module on the backend
- The app must be multi-tenant: each real estate agent only sees and manages their own data
- Prepare the app so it can later be deployed with frontend on Netlify and backend on Render
- Keep the code clean enough to export to VS Code and continue building
- Use TypeScript types across frontend and backend as much as possible

APP CONCEPT
A real estate agent signs up for the service, logs in, and completes an onboarding flow. They:
1. Build out their profile and structured business data
2. Choose AI voice preferences
3. Connect Google Calendar
4. Search for and purchase a Twilio number from inside the SaaS
5. Link that number to a Vapi assistant
6. Connect a real estate data source later (manual listings first, then support for IDX/MLS/RESO-compatible integrations)
7. Receive call summaries, lead records, appointments, and activity reporting through the dashboard
